## 1. 系统概述

EM-BOX 是面向具身智能数据采集行业（ITF/ITW/MWV 三类岗位）的 AI 招聘管理系统。核心能力包括：

| 能力 | 说明 |
|------|------|
| AI 候选人搜索 | 自然语言描述岗位需求，AI 自动匹配候选人 |
| AI 简历评分 | 上传简历自动解析 + 按岗位标准评分 |
| AI 面试 | 视频面试自动录音 → 语音转文字 → AI 评分 |
| 招聘漏斗 | 从入库到审批的完整 6 层追踪 |
| AI 代理 | 自动筛选、调度、匹配代理，24/7 运行 |
| 数据洞察 | 候选人来源分析、AI 代理效率、维度分析 |

### 系统架构

```
浏览器 → React 前端 (Vite) → Express 后端 (API) → PostgreSQL 数据库
                                      ↓
                              AI 模型服务（OpenAI / 智谱 / DeepSeek 等）
                                      ↓
                              MinerU 简历解析服务
```

### 快速启动

```bash
# 前端
npm run dev                    # 启动前端 http://localhost:3000

# 后端
cd server
npm run dev                    # 启动后端 http://localhost:4000
npm run migrate:seed           # 初始化数据库 + 测试数据
```

## 4. 候选人搜索

> 路径：`/search`

候选人搜索是系统的核心功能，支持 AI 自然语言搜索和传统筛选两种模式。

### AI 搜索

1. 在顶部搜索框输入自然语言描述，例如：
   > "有3年以上户外动作采集经验，会骑马或武术的候选人"
2. 点击「AI 搜索」或按 `Ctrl + Enter`
3. AI 会根据描述在候选人库中匹配最相关的结果

### 传统筛选

| 筛选器 | 说明 |
|--------|------|
| 项目 | 下拉选择项目，筛选该项目的候选人 |
| 岗位 | 下拉选择岗位，筛选匹配的候选人 |
| 时间范围 | 全部时间 / 本周 / 本月 / 自定义日期范围 |
| 等级 | 按评分等级筛选（S/A/B/C/D） |

### 候选人操作

| 操作 | 说明 |
|------|------|
| 查看详情 | 点击候选人卡片，弹出详情面板 |
| AI 评分 | 在详情面板中点击「AI 评分」，系统根据岗位标准自动评分 |
| 加入短名单 | 在详情面板中将候选人加入入围名单 |
| 发起外联 | 在详情面板中点击「发起外联」，通过邮件联系候选人 |
| 导出 CSV | 点击右上角「导出 CSV」按钮，下载候选人数据 |

### 简历导入

1. 点击「导入简历」按钮
2. 上传 PDF 或 Word 格式简历文件
3. 系统自动调用 MinerU 解析简历内容
4. 解析完成后候选人自动入库

## 6. 岗位标准配置

> 路径：`/positions/config`

配置每个岗位的评估标准和规则，AI 评分和筛选会基于这些配置。

### 配置项

| 配置 | 说明 |
|------|------|
| 基本信息 | 岗位名称、项目、需求人数、交付天数 |
| 画像规则 | 必备条件、加分条件、优先条件 |
| 评分规则 | 各维度权重和评分标准 |
| 等级规则 | 分数到等级的映射（如 90+ → S, 80-89 → A） |
| 基础分配置 | 各维度的基础分数设置 |

### 操作步骤

1. 左侧选择项目，系统列出该项目下的所有岗位
2. 点击岗位名称展开配置面板
3. 切换标签页编辑不同配置
4. 点击「保存」提交配置

### 导入面试评估文档

支持导入 MD 格式的结构化面试评估文档，系统自动解析并填充题目和评分配置。

## 7. AI 面试中心

> 路径：`/interviews/templates`

AI 面试中心包含 5 个子页面：

### 7.1 面试模板

> 路径：`/interviews/templates`

创建和管理面试模板。

- **创建模板**：设置模板名称、岗位、评分维度、等级规则
- **题目管理**：添加/编辑面试题目，支持分组、追问、评分标准
- **导入 MD**：从结构化文档批量导入题目

### 7.2 面试管理

> 路径：`/interviews/management`

管理面试会话。

| 操作 | 说明 |
|------|------|
| 创建会话 | 选择候选人 + 面试模板，创建面试会话 |
| 开始面试 | 点击播放按钮，进入 AI 视频面试流程 |
| 暂停/恢复 | 暂停或恢复正在进行的面试 |
| 删除会话 | 删除不需要的面试会话 |
| 导出结果 | 导出面试结果为 CSV |

### 7.3 AI 视频面试流程

1. 候选人进入面试页面
2. 系统按题目顺序逐题展示
3. 每道题自动录音（仅音频）
4. 录音完成后：
   - 音频 → Whisper 语音转文字
   - 文字 + 评分标准 → AI 评分
   - 保存每题评分和文字记录
5. 所有题目完成后自动汇总：
   - 计算总分和维度得分
   - 自动生成审批请求

### 7.4 面试结果

> 路径：`/interviews/results`

查看所有面试结果。

- 查看候选人姓名、岗位、分数、等级
- 点击展开查看维度得分和每题详情
- 支持导出 CSV

### 7.5 面试分析

> 路径：`/interviews/analytics`

面试数据分析看板。

| 分析维度 | 说明 |
|----------|------|
| 概览 | 总面试数、完成率、平均分、通过率 |
| 分数分布 | 各等级人数分布柱状图 |
| 通过率趋势 | 按月/周展示通过率变化 |
| 维度分析 | 各评分维度的平均得分，标红最弱维度 |
| 题目难度 | 每道题的平均分和低分率，标红最难题目 |

支持时间范围筛选：本周 / 本月 / 本季度 / 本年 / 全部。

## 18. 常见问题

### Q: 登录后页面空白或大量 401 错误

**原因**：Token 过期但会话标记未清除。

**解决**：打开浏览器控制台（F12 → Console），执行：
```javascript
localStorage.clear(); location.reload();
```

### Q: 创建用户时提示 CSRF 校验失败

**原因**：后端服务未重启，或前后端端口不匹配。

**解决**：
1. 重启后端服务：`cd server && npm run dev`
2. 确认 `.env` 中 `CORS_ORIGIN` 包含前端地址

### Q: AI 搜索没有结果

**原因**：数据库中没有候选人数据，或 AI 模型未配置。

**解决**：
1. 运行 seed 导入测试数据：`cd server && npm run seed`
2. 在设置中心或 AI 代理页面配置 AI 模型的 API Key

### Q: 简历解析失败

**原因**：MinerU API 未配置或服务不可用。

**解决**：
1. 在 `server/.env` 中配置 `MINERU_API_URL` 和 `MINERU_API_TOKEN`
2. 在集成管理页面检查 MinerU 连接状态

### Q: 面试评分失败

**原因**：AI 模型未配置或 Whisper 服务不可用。

**解决**：
1. 确认 `ai_model_configs` 表中有活跃的模型配置
2. 语音转文字需要 OpenAI 提供商（在 AI 配置中添加 OpenAI API Key）

### Q: 管理员如何重置用户密码

1. 登录管理员账号
2. 进入「设置中心」→「团队管理」标签页
3. 找到目标用户，点击钥匙图标
4. 输入新密码（至少 6 位），点击「确认重置」
