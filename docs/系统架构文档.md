# EM-BOX AI招聘平台 — 系统架构文档

> 版本: 2.2 | 更新日期: 2026-05-12

## 1. 技术栈总览

### 前端技术栈
- **框架**: React 19 + TypeScript 5 + Vite 6
- **样式**: Tailwind CSS 4
- **动画**: Motion (framer-motion)
- **图表**: Recharts
- **图标**: Lucide React
- **路由**: React Router v7
- **状态管理**: useState/useReducer (无全局状态库)

### 后端技术栈
- **运行时**: Node.js 18+
- **框架**: Express.js + TypeScript
- **数据库**: PostgreSQL 15+ (支持UUID, JSONB, TIMESTAMPTZ)
- **AI集成**: 4级PDF简历解析链路 (pdftotext → OCR → MinerU API → LLM Vision视觉识别), 可配置AI提供商 (OpenAI, Anthropic, Gemini, DeepSeek, Zhipu等), 支持多模态视觉模型
- **认证**: JWT (HS256算法), 基于角色的访问控制
- **构建工具**: tsx/tsup

### 开发工具链
- **构建**: Vite (前端), tsx/tsup (后端)
- **数据库迁移**: SQL migration files (001-015+)
- **API客户端**: fetchJson() + 自动JWT附加
- **模式切换**: USE_MOCK_API 环境变量控制

## 2. 系统架构图（文字描述）

EM-BOX AI招聘平台采用前后端分离的微服务架构，前端使用React单页应用，后端采用Express.js RESTful API服务。

### 核心业务模块
1. **候选人管理**: 简历导入、搜索、标签管理
2. **岗位配置**: 画像配置、评分标准、Grade Rules、基础分配置
3. **AI面试**: 面试模板、问题库、面试会话、结果分析
4. **审批流程**: 多级审批工作流
5. **人才库**: 短名单管理、人才池维护
6. **外联活动**: 营销活动、联系记录管理
7. **AI代理**: 智能化招聘流程管理
8. **数据分析**: 招聘效果分析、趋势预测

### 数据流向
- **前端** → **后端API** → **PostgreSQL数据库**
- **AI服务** (MinerU/OpenAI等) ←→ **后端AI模块** ←→ **前端应用**
- **简历解析**: 文件上传 → 4级降级解析 (pdftotext → OCR → MinerU API → LLM Vision) → 结构化存储
- **文件上传** → **后端存储** → **解析处理** → **数据库存储**

## 6. 前后端交互模式

### API通信模式
- **前端调用**: fetchJson(path, init) → 自动添加Authorization头部 → 发送到API_BASE_URL
- **后端返回**: snake_case JSON (PostgreSQL列名)
- **前端映射**: api.ts 将snake_case → camelCase使用映射函数
- **请求数据**: 前端发送camelCase → api.ts转换为snake_case用于POST/PATCH

### 数据映射策略
```typescript
// API响应映射 (snake_case → camelCase)
const snakeToCamel = (obj: any): any => {
  return Object.keys(obj).reduce((result, key) => {
    const camelKey = key.replace(/_([a-z])/g, (_, letter) => letter.toUpperCase());
    result[camelKey] = obj[key];
    return result;
  }, {});
};

// 请求数据映射 (camelCase → snake_case)
const camelToSnake = (obj: any): any => {
  return Object.keys(obj).reduce((result, key) => {
    const snakeKey = key.replace(/[A-Z]/g, letter => `_${letter.toLowerCase()}`);
    result[snakeKey] = obj[key];
    return result;
  }, {});
};
```

### 模式切换机制
- **开发模式** (USE_MOCK_API=true): 所有数据存储在localStorage，无需后端
- **生产模式** (USE_MOCK_API=false): 连接真实后端API
- **API客户端**自动根据环境变量切换行为

### 错误处理机制
- **网络错误**: 自动重试机制
- **认证错误**: 自动跳转登录页面
- **业务错误**: 友好的错误提示
- **系统错误**: 统一的错误日志记录
