## 1. 测试环境搭建

### 1.1 开发环境要求
- 操作系统: Windows 10/11, macOS, Linux
- 浏览器: Chrome 90+, Firefox 88+, Safari 14+
- Node.js: v16.0+
- npm: 8.0+

### 1.2 环境配置步骤

#### 前端环境
```bash
# 进入项目目录
cd /Users/tree/Desktop/Ops\ Mind\ Ai/Trai-main

# 安装依赖
npm install

# 启动前端开发服务器（端口5173）
npm run dev
```

#### 后端环境
```bash
# 创建环境变量文件
cp .env.example .env

# 配置数据库连接
# 编辑 .env 文件，设置数据库连接信息

# 安装后端依赖
cd server
npm install

# 启动后端服务器（端口4000）
npm run dev
```

#### 模拟模式配置
```bash
# 设置环境变量使用模拟API
export VITE_USE_MOCK_API=true

# 启动前端
npm run dev
```

#### 真实模式配置
```bash
# 设置环境变量使用真实API
export VITE_USE_MOCK_API=false

# 确保后端服务器正在运行
# 确保PostgreSQL数据库服务已启动
```

## 3. 测试流程（7个核心流程）

### 3.1 流程1: 项目管理

#### 3.1.1 登录系统
1. 打开浏览器访问 `http://localhost:5173`
2. 输入管理员账号：`admin@embox.com` / `admin123`
3. 点击"登录"按钮
4. 验证成功跳转到仪表板

#### 3.1.2 创建新项目
1. 在左侧导航栏点击"项目管理"
2. 点击"新建项目"按钮
3. 填写项目信息：
   - 项目名称：`测试项目001`
   - 城市：`北京`
   - 项目经理：`张经理`
   - 开始日期：`2026-05-01`
   - 结束日期：`2026-05-31`
4. 点击"保存"按钮
5. 验证项目成功创建并出现在项目列表中

#### 3.1.3 编辑项目详情
1. 在项目列表中找到刚创建的项目
2. 点击"编辑"按钮
3. 修改项目信息：
   - 将项目名称改为：`测试项目001-已修改`
   - 将项目经理改为：`李经理`
4. 点击"保存"按钮
5. 验证项目信息已更新

#### 3.1.4 修改项目状态
1. 在项目列表中选择一个项目
2. 点击"状态"下拉菜单
3. 依次测试以下状态切换：
   - `规划中` → `进行中` → `已完成` → `已暂停`
4. 验证状态更新正确且时间戳记录完整

#### 3.1.5 验证项目显示
1. 返回仪表板页面
2. 验证新建的项目出现在"项目统计"区域
3. 检查项目数量、状态分布等统计信息是否正确

#### 3.1.6 测试数据验证
- 必填字段验证：项目名称、城市、日期为必填项
- 日期验证：结束日期不能早于开始日期
- 名称验证：项目名称不能重复
- 字符长度验证：各字段长度限制

### 3.2 流程2: 岗位配置

#### 3.2.1 进入岗位配置页面
1. 在左侧导航栏点击"岗位标准配置"
2. 验证页面加载完成，显示岗位列表

#### 3.2.2 创建新岗位
1. 点击"新建岗位"按钮
2. 填写基本信息：
   - 岗位名称：`AI训练师`
   - 岗位描述：`负责AI模型训练和优化`
   - 所属部门：`技术部`
   - 工作地点：`北京`
3. 点击"保存基本信息"

#### 3.2.3 配置画像配置（Profile Rules）
1. 切换到"画像配置"标签页
2. 点击"添加规则"按钮
3. 依次添加以下规则：
   ```
   规则1:
   - 关键词：计算机操作熟练
   - 同义词：电脑操作熟练,Office办公软件熟练
   - 类别：基础能力
   
   规则2:
   - 关键词：数据分析
   - 同义词：数据处理,数据挖掘,统计分析
   - 类别：专业技能
   
   规则3:
   - 关键词：AI训练经验
   - 同义词：深度学习训练,机器学习训练
   - 类别：工作经验
   ```
4. 验证规则添加成功，支持编辑和删除操作

#### 3.2.4 配置评分标准（Scoring Rules）
1. 切换到"评分标准"标签页
2. 点击"添加维度"按钮
3. 依次添加以下维度：
   ```
   维度1：
   - 维度名称：经验对口度
   - 权重：30%
   - 关键字：AI训练,深度学习,机器学习
   - 匹配模式：any
   
   维度2：
   - 维度名称：技能匹配度
   - 权重：40%
   - 关键字：Python,机器学习框架,数据处理
   - 匹配模式：any
   
   维度3：
   - 维度名称：教育背景
   - 权重：20%
   - 关键字：本科,硕士,计算机相关专业
   - 匹配模式：any
   
   维度4：
   - 维度名称：项目经验
   - 权重：10%
   - 关键字：AI项目,机器学习项目
   - 匹配模式：any
   ```
4. 验证维度添加成功，权重总和应为100%

#### 3.2.5 配置Grade Rules
1. 切换到"Grade Rules"标签页
2. 点击"添加档位"按钮
3. 依次添加以下档位：
   ```
   A档：
   - 最低分：80
   - 最高分：100
   - 操作建议：优先录用
   
   B档：
   - 最低分：60
   - 最高分：79
   - 操作建议：考虑录用
   
   C档：
   - 最低分：40
   - 最高分：59
   - 操作建议：谨慎考虑
   
   D档：
   - 最低分：0
   - 最高分：39
   - 操作建议：不建议录用
   ```
4. 验证档位添加成功，分数区间无重叠

#### 3.2.6 配置基础分
1. 切换到"基础分配置"标签页
2. 设置基础信息：
   - 基础分值：50分
   - 触发条件：必备项审核通过
   - 必备项数量：4
   - 必备项列表：
     - 电脑操作熟练
     - 排班出勤
     - 团队协作
     - 沟通能力
3. 点击"保存"按钮
4. 验证配置保存成功

#### 3.2.7 完整保存和验证
1. 返回基本信息页面
2. 点击"保存完整配置"按钮
3. 验证配置成功保存提示
4. 重新进入页面验证所有配置是否保持正确
5. 导入测试简历验证评分功能是否正常工作

```
## 个人信息
- 姓名：张三
- 年龄：28岁
- 学历：本科
- 专业：计算机科学与技术
```

#### 3.3.4 选择岗位进行评分

2. 点击"开始评分"按钮
3. 等待AI评分完成
4. 验证评分结果是否显示

#### 3.3.5 验证AI评分结果
1. 检查以下评分结果字段：
   - 总分：预期范围0-100分
   - 等级：A/B/C/D档位
   - 各维度评分：与配置的权重和评分项一致
   - 匹配关键词：高亮显示匹配的关键词
   - 缺失关键词：显示未匹配的关键词
2. 验证评分逻辑的合理性
3. 记录详细的评分结果用于对比分析

#### 3.3.6 查看候选人详情

#### 3.3.7 测试下载功能

#### 3.3.8 添加到入围名单

### 6.2 缺陷严重程度定义

#### 高严重程度
- **描述**: 系统核心功能完全无法使用
- **影响**: 导致主要业务流程中断
- **示例**: 
  - 用户无法登录
  - 数据无法保存
  - 关键模块崩溃
- **处理时间**: 24小时内修复

#### 中等严重程度
- **描述**: 功能部分受限，或有明显缺陷
- **影响**: 影响用户体验，但不中断主要流程
- **示例**:
  - 页面显示异常
  - 数据计算错误
  - 性能明显下降
- **处理时间**: 3个工作日内修复

#### 低严重程度
- **描述**: 界面问题或次要功能缺陷
- **影响**: 不影响核心功能，仅影响美观或便利性
- **示例**:
  - 文字显示不完美
  - 按钮位置不佳
  - 提示信息不够友好
- **处理时间**: 下个版本修复

### 6.4 缺陷报告跟踪流程

3. **分配开发**: 分配给相应的开发人员
4. **修复开发**: 开发人员进行修复
5. **验证测试**: 测试人员验证修复结果
6. **关闭缺陷**: 确认修复完成，关闭缺陷报告
7. **回归测试**: 确保修复没有引入新问题

1. **测试优先级**: 按照功能重要性安排测试顺序，先测试核心业务流程，再测试辅助功能

3. **测试数据**: 准备足够的测试数据，覆盖各种业务场景
4. **测试覆盖**: 确保测试用例覆盖所有功能模块和边界情况
5. **测试报告**: 及时记录测试结果，跟踪缺陷修复状态

1. **自动化测试**: 建议增加自动化测试脚本，提高测试效率

### 8.3 风险提示
