# EM-BOX AI招聘平台 — 产品需求文档 (PRD)

> 版本: 1.2 | 更新日期: 2026-05-26

## 1. 产品概述

EM-BOX是一个专注于 Ego · UMI · 真机/本体数据 · 仿真/合成数据 四类数据采集岗位的AI驱动招聘管理平台。通过智能化的简历解析、匹配评分和自动化外联流程，为Embodied AI数据采集项目提供高效的人才筛选和管理解决方案。

### 1.1 产品愿景
打造专业化的AI招聘平台，为Embodied AI行业提供精准、高效的人才匹配和管理服务。

### 1.2 产品目标
- 提升简历筛选效率80%
- 降低人工匹配成本60%
- 提高候选人转化率40%
- 实现招聘流程全数字化管理

### 1.3 核心价值主张
- **AI智能匹配**：基于岗位画像和评分标准的智能算法
- **自动化流程**：从简历导入到外联的自动化管理
- **数据驱动决策**：完整的招聘数据分析和洞察
- **专业化定制**：针对Embodied AI行业特点的深度定制

## 2. 目标用户与使用场景

### 2.1 用户角色定义

#### 招募专员 (Recruiter)
**职责描述**：负责候选人搜索、简历筛选、外联跟进和面试管理
**技能要求**：熟悉招聘流程，具备沟通协调能力，了解Embodied AI行业背景

**主要使用场景**：
- 日常批量简历导入和AI评分
- 根据评分结果筛选入围候选人
- 执行AI外联跟进和面试邀请
- 管理候选人全流程状态

#### 管理员 (Admin)
**职责描述**：系统配置管理、岗位标准制定、团队权限管理、审批监督
**技能要求**：具备系统管理能力，熟悉业务流程，有团队管理经验

**主要使用场景**：
- 配置和管理岗位评分标准
- 设置团队权限和工作流程
- 监控招聘数据和团队绩效
- 审批重要决策和结果

#### 候选人 (Candidate)
**职责描述**：参与AI面试，提交个人资料，获取招聘结果
**技能要求**：具备基本的计算机操作能力，理解AI面试流程

**主要使用场景**：
- 在线提交简历和个人信息
- 参与AI视频面试
- 查看面试结果和反馈
- 接收录用通知

#### 访问者 (Viewer)
**职责描述**：查看招聘数据，监控进度，提供决策支持
**技能要求**：具备数据分析基础，了解招聘业务

**主要使用场景**：
- 查看招聘进度和统计数据
- 监控团队绩效和效率
- 获取招聘分析报告
- 参与招聘决策讨论

## 3. 核心业务术语

### 3.1 技术术语
- **Ego (Egocentric Video)**：第一人称视角视频数据采集，以自身视角记录操作过程
- **UMI (Universal Manipulation Interface)**：通用操作接口，用于标准化操作数据采集
- **真机/本体数据**：在真实机器人或设备上采集的实际运行数据
- **仿真/合成数据**：通过仿真环境生成的合成数据，用于训练和测试
- **Fit Score**：AI匹配分数，0-100分的候选人匹配度评估
- **AI外联**：基于AI算法的自动化候选人联系和跟进

### 3.2 业务术语
- **岗位画像**：定义岗位所需的关键技能和特征
- **评分标准**：评估候选人匹配度的具体维度和权重
- **入围名单**：经过初步筛选的高优先级候选人集合
- **外联序列**：按优先级排序的候选人联系计划
- **审批中心**：面试结果和录用决策的审批流程
- **人才库**：历史候选人的分类存储和管理系统

### 3.3 数据术语
- **匹配率**：简历关键词与岗位要求的匹配百分比
- **转化率**：从候选人到录用的转化比例
- **简历质量分数**：简历完整性和相关性的综合评分
- **面试完成率**：AI面试的完成比例统计

## 4. 业务流程

### 4.1 核心业务流程

#### 项目管理流程
```
项目创建 → 项目配置 → 岗位定义 → 人员需求确认 → 招聘计划制定
```

#### 招聘执行流程
```
简历收集 → AI简历解析 → 匹配评分 → 人工审核 → 入围名单确认 → 外联计划制定 → AI面试安排 → 面试执行 → 结果评估 → 最终决策
```

#### 审批流程
```
面试结果确认 → 录用条件评估 → 薪资谈判 → 最终审批 → 录用通知 → 入职准备
```

### 4.2 关键业务规则

#### 简历评分规则
- 必须包含的核心关键词：完整匹配得满分，部分匹配按比例计分
- 加分项：相关专业经验、证书认证、特殊技能
- 减分项：经验不匹配、技能缺失、项目经验不符

#### 外联优先级规则
- 按Fit Score降序排列
- 按岗位紧急程度分组
- 考虑候选人可用性
- 基于历史外联成功率

#### 审批权限规则
- 普通审批：团队主管
- 重要审批：部门经理
- 特殊审批：HR总监
- 最终审批：高管层
