## 1. 环境要求

### 系统要求

- **操作系统**: Linux (推荐 Ubuntu 20.04+), macOS 11+, 或 Windows 10+
- **内存**: 最少 4GB，推荐 8GB+
- **存储**: 最少 10GB 可用空间
- **CPU**: 双核以上，推荐四核

### 软件要求

- **Node.js**: 18.x 或更高版本
- **PostgreSQL**: 15.x 或更高版本
- **pdftoppm**: PDF转图片工具 (poppler-utils 包)，LLM Vision简历解析必需
- **tesseract**: OCR引擎，需安装中文语言包 `tesseract-lang-chi-sim`，扫描版PDF解析必需
- **npm** 或 **pnpm**: 包管理器
- **现代浏览器**: Chrome 90+, Edge 90+, Firefox 88+, Safari 14+

### 开发工具推荐

- **代码编辑器**: Visual Studio Code
- **数据库工具**: DBeaver, pgAdmin, 或 DataGrip
- **API测试工具**: Postman, Insomnia, 或 curl
- **版本控制**: Git

## 2. 本地开发环境搭建

### 2.1 安装依赖

```bash
# 克隆项目
git clone <repository-url>
cd Trai-main

# 安装前端依赖
npm install

# 安装后端依赖
cd server
npm install
cd ..
```

### 2.2 数据库安装与配置

#### Linux/macOS

```bash
# PostgreSQL 安装
sudo apt update
sudo apt install postgresql postgresql-contrib

# PDF解析工具 (LLM Vision简历解析必需)
sudo apt install poppler-utils        # 包含 pdftoppm
sudo apt install tesseract-ocr         # OCR引擎
sudo apt install tesseract-ocr-chi-sim # 中文语言包
```

macOS:
```bash
brew install poppler tesseract tesseract-lang
```

# 启动服务
sudo systemctl start postgresql
sudo systemctl enable postgresql

# 创建数据库和用户
sudo -u postgres createdb embox
sudo -u postgres psql -c "CREATE USER embox_user WITH PASSWORD 'your_password';"
sudo -u postgres psql -c "GRANT ALL PRIVILEGES ON DATABASE embox TO embox_user;"
```

#### Windows

```powershell
# 使用 Chocolatey 安装
choco install postgresql

# 或者手动下载安装
# https://www.postgresql.org/download/windows/

# 设置 PostgreSQL 密码
psql -U postgres -c "ALTER USER postgres WITH PASSWORD 'your_password';"
createdb embox
psql -U postgres -c "CREATE USER embox_user WITH PASSWORD 'your_password';"
psql -U postgres -c "GRANT ALL PRIVILEGES ON DATABASE embox TO embox_user;"
```

### 2.3 数据库迁移

```bash
cd server
npx tsx src/db/run-migrations.ts
```

### 2.4 环境变量配置

创建并配置前端环境变量文件：
```bash
# 创建 .env 文件
cat > .env << EOF
VITE_USE_MOCK_API=false
VITE_API_BASE_URL="http://localhost:4000"
MINERU_API_TOKEN=your_mineru_token_here
EOF
```

创建并配置后端环境变量文件：
```bash

JWT_SECRET=your-very-secure-secret-key-here

```

### 2.5 启动服务

```bash
# 启动后端服务（端口 4000）
cd server
npm run dev

# 新终端启动前端服务（端口 3000）
cd ..
npm run dev
```

## 4. 环境变量配置

### 4.1 前端环境变量

文件位置：`/Trai-main/.env`

```env
# API 模式配置
VITE_USE_MOCK_API=false                    # true=mock模式，false=真实API
VITE_API_BASE_URL="http://localhost:4000"   # 后端服务URL
MINERU_API_TOKEN=your_mineru_token_here     # MinerU简历解析Token

# 可选配置
VITE_APP_TITLE="EM-BOX AI招聘平台"         # 应用标题
VITE_APP_VERSION="1.0.0"                  # 应用版本
```

### 4.2 后端环境变量

文件位置：`/Trai-main/server/.env`

```env
# 数据库配置
DATABASE_URL=postgresql://user:password@localhost:5432/embox
DATABASE_SSL=false                        # 生产环境建议启用SSL
DATABASE_POOL_SIZE=10                      # 连接池大小
DATABASE_CONNECTION_TIMEOUT=30000         # 连接超时(毫秒)

# JWT 配置
JWT_SECRET=your-super-secret-jwt-key-here
JWT_EXPIRES_IN=24h                         # Token过期时间
JWT_REFRESH_EXPIRES_IN=7d                  # Refresh Token过期时间

# 服务器配置
PORT=4000
NODE_ENV=development                       # development/production
HOST=0.0.0.0                              # 监听地址

# 安全配置
CORS_ORIGIN=http://localhost:3000         # CORS允许的源
BCRYPT_ROUNDS=12                          # 密码加密轮数

# 文件上传配置
UPLOAD_DIR=uploads                         # 上传目录
MAX_FILE_SIZE=10485760                    # 最大文件大小(10MB)

# 日志配置
LOG_LEVEL=info                            # debug/info/warn/error
LOG_DIR=logs                              # 日志目录

# 外部服务配置
MINERU_API_URL=https://api.mineru.com     # MinerU API地址
OPENAI_API_KEY=your-openai-key           # OpenAI API Key（可选）
```

## 5. 启动与停止服务

### 5.2 生产环境

#### 使用 PM2

```bash
# 安装 PM2
npm install -g pm2

# 启动后端
cd server
pm2 start dist/server.js --name "embox-api"

# 启动前端（需要先构建）
cd ..
npm run build
pm2 start dist/index.html --name "embox-frontend"

# 保存 PM2 配置
pm2 save
pm2 startup

# 查看进程状态
pm2 status

# 查看日志
pm2 logs embox-api
pm2 logs embox-frontend
```

#### 使用 Docker Compose

```yaml
# docker-compose.yml
version: '3.8'

services:
  frontend:
    build: .
    ports:
      - "3000:3000"
    environment:
      - VITE_API_BASE_URL=http://backend:4000
      - VITE_USE_MOCK_API=false
    depends_on:
      - backend

  backend:
    build: ./server
    ports:
      - "4000:4000"
    environment:
      - DATABASE_URL=postgresql://postgres:password@db:5432/embox
      - JWT_SECRET=your-jwt-secret
      - NODE_ENV=production
    depends_on:
      - db

  db:
    image: postgres:15
    environment:
      - POSTGRES_DB=embox
      - POSTGRES_USER=postgres
      - POSTGRES_PASSWORD=password
    volumes:
      - postgres_data:/var/lib/postgresql/data
    ports:
      - "5432:5432"

volumes:
  postgres_data:
```

启动服务：
```bash
docker-compose up -d
```

## 6. Mock模式 vs 真实模式

### 6.1 Mock模式

**特性：**
- 所有数据存储在浏览器 localStorage
- 无需后端服务
- 适合UI测试和演示

**配置：**
```env
VITE_USE_MOCK_API=true
```

**适用场景：**
- 前端开发初期
- UI/UX 设计阶段
- 演示和原型制作
- 网络条件受限环境

**局限性：**
- 无法进行真实API测试
- 无法保存持久化数据
- 无法测试真实业务逻辑

### 6.2 真实模式

**特性：**
- 连接真实后端API
- 数据持久化存储
- 完整的业务功能

**配置：**
```env
VITE_USE_MOCK_API=false
VITE_API_BASE_URL="http://localhost:4000"
```

**适用场景：**
- 正常开发调试
- 功能测试
- 性能测试
- 生产环境

### 6.3 模式切换

在开发过程中切换模式：

```bash
# 切换到Mock模式
echo "VITE_USE_MOCK_API=true" > .env
npm run dev

# 切换到真实模式
echo "VITE_USE_MOCK_API=false" > .env
echo "VITE_API_BASE_URL=http://localhost:4000" >> .env
npm run dev
```

### 6.4 模式验证

检查当前模式：
```bash
# 查看环境变量
cat .env | grep VITE_USE_MOCK_API

# 测试API响应
curl http://localhost:4000/api/health
```

## 9. 数据备份与恢复

### 9.1 数据库备份

#### 完整备份
```bash
# 创建备份目录
mkdir -p backups

# 执行备份
pg_dump -U postgres -h localhost -d embox > backups/embox_$(date +%Y%m%d_%H%M%S).sql

# 压缩备份
gzip backups/embox_$(date +%Y%m%d_%H%M%S).sql
```

#### 增量备份
```bash
# 使用 pg_dump 进行增量备份
pg_dump -U postgres -h localhost -d embox --data-only > backups/embox_incremental_$(date +%Y%m%d_%H%M%S).sql
```

#### 自动备份脚本
```bash
#!/bin/bash
# backup.sh

BACKUP_DIR="/var/backups/embox"
DATE=$(date +%Y%m%d_%H%M%S)
DB_NAME="embox"
DB_USER="postgres"

# 创建备份目录
mkdir -p $BACKUP_DIR

# 执行备份
pg_dump -U $DB_USER -h localhost -d $DB_NAME > $BACKUP_DIR/embox_$DATE.sql

# 保留最近7天的备份
find $BACKUP_DIR -name "*.sql" -mtime +7 -delete

# 压缩旧备份
find $BACKUP_DIR -name "*.sql" -mtime +1 -exec gzip {} \;
```

### 9.2 数据库恢复

#### 恢复完整备份
```bash
# 解压备份
gunzip backups/embox_20240506_120000.sql.gz

# 停止数据库服务
sudo systemctl stop postgresql

# 恢复数据库
psql -U postgres -d embox < backups/embox_20240506_120000.sql

# 启动数据库服务
sudo systemctl start postgresql
```

#### 恢复特定表
```bash
# 恢复单个表
pg_dump -U postgres -h localhost -d embox -t candidates > candidates_backup.sql
psql -U postgres -d embox < candidates_backup.sql
```

const fs = require('fs');
const path = require('path');

function rotateLogs() {
  const logDir = 'logs';
  const today = new Date().toISOString().split('T')[0];
  
  // 创建今天的日志文件
  const todayLogFile = path.join(logDir, `application-${today}.log`);
  
  if (!fs.existsSync(todayLogFile)) {
    // 关闭当前日志文件
    logger.transports.forEach(transport => {
      if (transport.filename && transport.filename.includes('application')) {
        transport.end();
      }
    });

    const transport = new winston.transports.File({
      filename: todayLogFile,
      maxsize: 50 * 1024 * 1024, // 50MB
      maxFiles: 10
    });
    
    logger.add(transport);
  }
}

### 15.6 PDF 解析架构变化

**重要变化：** 生产环境 (Vercel + Supabase) 中，PDF 简历解析由前端直接调用 MinerU API，不再经过后端。

**原因：** 后端 `pdfProxy.ts` 依赖 `pdftotext` 和 `tesseract`（需要 `exec()`），而 Supabase Edge Functions 不支持 `exec()`。

**生产流程：**
```
用户上传 PDF → 前端 mineruClient.ts → 直接 POST 到 MinerU API → 返回解析结果
```

**开发流程（保持不变）：**
```
用户上传 PDF → 前端 → 后端 /api/mineru/file_parse → pdftotext/OCR/MinerU/LLM Vision
```
