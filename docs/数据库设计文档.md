# EM-BOX AI招聘平台 — 数据库设计文档

> 版本: 2.0 | 更新日期: 2026-05-06

## 1. 概述

### 数据库基本信息
- **数据库类型**: PostgreSQL 14+
- **字符集**: UTF-8
- **时区**: UTC (TIMESTAMPTZ类型)
- **主键策略**: UUID (gen_random_uuid())
- **命名规范**: snake_case

### 设计原则
1. **数据分离**: 配置数据与运行数据分离存储
2. **版本控制**: 关键配置支持版本化追踪
3. **审计追踪**: 重要操作都有时间戳记录
4. **关系完整性**: 外键约束保证数据一致性
5. **性能优化**: 合理的索引设计提升查询效率

## 2. ER关系图（文字描述）

### 核心实体关系
- **用户系统**: users → roles (多对多关系通过user_roles)
- **项目架构**: users → projects → positions (层级结构)
- **候选人管理**: positions → candidates (一对多关系)
- **岗位配置**: positions → position_details (一对一关系，包含JSON配置)
- **面试流程**: positions → interview_templates → interview_questions → interview_sessions → interview_results
- **审批流程**: users → approval_requests (审批链)
- **外联管理**: positions → outreach_campaigns → outreach_records
- **AI系统**: ai_model_configs (全局配置) → agents (项目级AI代理)

### 数据流向
1. 用户创建项目 → 在项目中配置岗位 → 岗位包含详细的评分和画像配置
2. 候选人通过不同渠道进入系统 → 系统自动解析简历 → 根据岗位配置进行评分
3. 面试模板管理 → 面试会话执行 → 生成面试结果 → 审批流程
4. 外联活动管理 → 联系记录追踪 → 短名单管理

## 3. 表结构详细定义

### 3.1 用户管理表

#### users
```sql
CREATE TABLE users (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name          VARCHAR(200) NOT NULL,
  email         VARCHAR(320) UNIQUE NOT NULL,
  password_hash VARCHAR(200) NOT NULL,
  role          VARCHAR(50) NOT NULL DEFAULT 'viewer',
  avatar        TEXT,
  phone         VARCHAR(50),
  department    VARCHAR(200),
  status        VARCHAR(20) NOT NULL DEFAULT 'active'
                CHECK (status IN ('active', 'inactive')),
  last_login_at TIMESTAMPTZ,
  created_at    TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at    TIMESTAMPTZ NOT NULL DEFAULT now()
);
```

**字段说明**:
- `id`: 用户唯一标识
- `name`: 用户姓名
- `email`: 用户邮箱（唯一）
- `password_hash`: 密码哈希值
- `role`: 用户角色（admin, recruiter, interviewer, viewer）
- `avatar`: 头像URL
- `phone`: 联系电话
- `department`: 所属部门
- `status`: 用户状态
- `last_login_at`: 最后登录时间
- `created_at/updated_at`: 创建和更新时间

**索引**:
```sql
CREATE INDEX idx_users_email ON users(email);
CREATE INDEX idx_users_role  ON users(role);
```

**字段说明**:

- `category`: 岗位类别

- `status`: 岗位状态
- `description`: 岗位描述
- `created_by`: 岗位创建人
- `required_count`: 需求数量

- `created_at/updated_at`: 创建和更新时间

### 3.4 候选人管理表

```sql
CREATE TABLE candidates (

  name                  VARCHAR(200) NOT NULL,

  original_file_name    VARCHAR(500),
  score_total           NUMERIC(5,2),
  grade                 VARCHAR(10),
  created_at            TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at            TIMESTAMPTZ NOT NULL DEFAULT now()
);
```

**字段说明**:
- `id`: 候选人唯一标识
- `name`: 候选人姓名
- `email`: 候选人邮箱（唯一）
- `phone`: 联系电话
- `location`: 所在地
- `source`: 来源渠道
- `project_id/position_id`: 所属项目和岗位
- `raw_resume_md`: 简历原文（Markdown格式）
- `parsed_info`: 解析后的简历信息（JSON）
- `original_file_base64`: 原始文件Base64编码
- `original_file_name`: 原始文件名
- `score_total/grade`: 总分和等级
- `created_at/updated_at`: 创建和更新时间

**约束**:
```sql
-- 唯一邮箱约束
ALTER TABLE candidates ADD CONSTRAINT uq_candidates_email UNIQUE (email);

-- 组合唯一约束（姓名+电话）
CREATE UNIQUE INDEX uq_candidates_name_phone ON candidates (name, phone) WHERE phone IS NOT NULL;

-- 更新时间触发器
CREATE OR REPLACE FUNCTION update_candidate_timestamp()
RETURNS TRIGGER AS $$
BEGIN
  NEW.updated_at = now();
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER trg_candidates_updated_at
  BEFORE UPDATE ON candidates
  FOR EACH ROW EXECUTE FUNCTION update_candidate_timestamp();
```

**索引**:
```sql
CREATE INDEX idx_candidates_name      ON candidates(name);
CREATE INDEX idx_candidates_position  ON candidates(position_id);
CREATE INDEX idx_candidates_project   ON candidates(project_id);
CREATE INDEX idx_candidates_grade     ON candidates(grade);
```

## 6. 数据库迁移规范

### 6.1 迁移文件命名规范
- 使用数字前缀排序：`001_`, `002_`, `003_`, ...
- 文件名格式：`{序号}_{动作}_{表名}.sql`
- 示例：`001_create_users.sql`, `002_create_projects.sql`

### 6.2 迁移文件内容规范
```sql
-- 迁移文件头部注释
-- Migration: {序号}
-- Description: {迁移描述}
-- Author: {开发者姓名}
-- Date: {创建日期}

-- 迁移SQL语句
CREATE TABLE ...;

-- 迁移完成注释
-- Migration completed successfully
```

### 6.3 迁移执行原则
1. **原子性**: 每个迁移文件必须能够独立执行
2. **可逆性**: 重要迁移应有回滚方案
3. **兼容性**: 考虑现有数据的兼容性
4. **性能**: 避免长时间锁表操作

### 6.4 版本管理
- 每次部署执行迁移时记录版本信息
- 使用数据库迁移工具（如Prisma、Liquibase）管理迁移历史
- 重要迁移前进行备份

## 7. 常用查询示例

### 7.1 项目相关查询

```sql
-- 查询项目列表（包含岗位数量）
SELECT 
  p.*,
  COUNT(pos.id) as position_count
FROM projects p
LEFT JOIN positions pos ON p.id = pos.project_id
GROUP BY p.id;

-- 查询项目进度
SELECT 
  p.name,
  p.city,
  p.manager,
  p.progress,
  COUNT(c.id) as candidate_count
FROM projects p
LEFT JOIN candidates c ON p.id = c.project_id
GROUP BY p.id;
```

### 7.2 岗位相关查询

```sql
-- 查询岗位详情（包含配置信息）
SELECT 
  p.*,
  pd.profile,
  pd.scoring_rules,
  pd.grade_rules
FROM positions p
LEFT JOIN position_details pd ON p.id = pd.position_id
WHERE p.id = '岗位UUID';

-- 查询岗位候选人统计
SELECT 
  p.name,
  COUNT(c.id) as candidate_count,
  COUNT(CASE WHEN c.score_total >= 80 THEN 1 END) as high_score_count,
  COUNT(CASE WHEN c.score_total < 60 THEN 1 END) as low_score_count
FROM positions p
LEFT JOIN candidates c ON p.id = c.position_id
GROUP BY p.id;
```

### 7.3 候选人相关查询

```sql
-- 查询候选人列表（包含面试状态）
SELECT 
  c.*,
  it.name as interview_template_name,
  is.status as interview_status,
  ir.total_score as interview_score
FROM candidates c
LEFT JOIN interview_sessions is ON c.id = is.candidate_id
LEFT JOIN interview_templates it ON is.template_id = it.id
ORDER BY c.created_at DESC;

-- 查询候选人评分详情
SELECT 
  c.name,
  c.email,
  c.score_total,
  c.grade,
  pd.profile_rules,
  pd.scoring_rules
FROM candidates c
LEFT JOIN positions pos ON c.position_id = pos.id
LEFT JOIN position_details pd ON pos.id = pd.position_id
WHERE c.id = '候选人UUID';
```

### 7.4 面试相关查询

```sql
-- 查询面试统计
SELECT 
  EXTRACT(MONTH FROM ir.interview_date) as month,
  COUNT(*) as interview_count,
  AVG(ir.total_score) as avg_score,
  COUNT(CASE WHEN ir.grade = 'A' THEN 1 END) as a_grade_count
FROM interview_results ir
GROUP BY EXTRACT(MONTH FROM ir.interview_date)
ORDER BY month;

-- 查面试结果详情
SELECT 
  ir.*,
  c.name as candidate_name,
  p.name as position_name,
  it.name as template_name
FROM interview_results ir
LEFT JOIN candidates c ON ir.candidate_id = c.id
LEFT JOIN positions p ON ir.position_id = p.id
LEFT JOIN interview_templates it ON ir.template_id = it.id
WHERE ir.id = '面试结果UUID';
```

### 7.5 审批相关查询

```sql
-- 查询待审批列表
SELECT 
  ar.*,
  c.name as candidate_name,
  p.name as position_name,
  u.name as requester_name
FROM approval_requests ar
LEFT JOIN candidates c ON ar.candidate_id = c.id
LEFT JOIN positions p ON ar.position_id = p.id
LEFT JOIN users u ON ar.requester_name = u.email
WHERE ar.status = 'pending'
ORDER BY ar.created_at DESC;

-- 查询审批统计
SELECT 
  EXTRACT(MONTH FROM ar.created_at) as month,
  COUNT(*) as total_requests,
  COUNT(CASE WHEN ar.status = 'approved' THEN 1 END) as approved_count,
  COUNT(CASE WHEN ar.status = 'rejected' THEN 1 END) as rejected_count
FROM approval_requests ar
GROUP BY EXTRACT(MONTH FROM ar.created_at)
ORDER BY month;
```

### 7.6 外联相关查询

```sql
-- 查询外联活动效果
SELECT 
  oc.*,
  COUNT(orr.id) as sent_count,
  COUNT(CASE WHEN orr.status = 'replied' THEN 1 END) as replied_count,
  ROUND(
    COUNT(CASE WHEN orr.status = 'replied' THEN 1 END) * 100.0 / 
    NULLIF(COUNT(orr.id), 0), 2
  ) as reply_rate
FROM outreach_campaigns oc
LEFT JOIN outreach_records orr ON oc.id = orr.candidate_id
GROUP BY oc.id;

-- 查询候选人外联记录
SELECT 
  orr.*,
  c.name as candidate_name,
  p.name as position_name
FROM outreach_records orr
LEFT JOIN candidates c ON orr.candidate_id = c.id
LEFT JOIN positions p ON orr.position_id = p.id
WHERE orr.candidate_id = '候选人UUID'
ORDER BY orr.created_at DESC;
```
