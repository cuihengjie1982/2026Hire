# EM-BOX AI招聘平台 — 更新日志

所有重要功能变更均记录在此文档中。格式基于 [Keep a Changelog](https://keepachangelog.com/zh-CN/)。

## [1.3.0] - 2026-05-18

### 新增
- **Edge Functions 双路由架构**：新增 `embox-api`（verify_jwt=false）和 `index`（verify_jwt=true）两个入口，支持更灵活的权限控制
  - `embox-api`：主路由，处理 settings/users、AI config、cross-table ops、agent-executor、mineru-proxy、notifications、sms-gateway、analytics
  - `index`：旧路由，处理 ai-proxy、interview-scoring、candidate-ops 等
  - 前端推荐使用 `efetch` 函数直接调用 `/functions/v1/embox-api/<path>`

### 变更
- **登录页文案更新**：`ITF · ITW · MWV` → `Ego · UMI · 真机/本体数据 · 仿真/合成数据`，`三类数据采集岗位` → `四类数据采集岗位`
- **登录页标题**：`My Google AI Studio App` → `EM-BOX AI招聘智能助手`
- **Settings API** 全面迁移至 `embox-api` Edge Function，不再使用 `invokeEdgeFunction`
- **项目管理统计** 移除硬编码 fallback 值（406、58、6），mock 模式正确返回 0
- **两套后端系统** 文档化：本地 Express（:4000）vs 生产 Supabase Edge Functions

## [1.2.0] - 2026-05-16

### 新增
- **AI面试评分管线**：完整的音频录制→Whisper转录→LLM结构化评分→聚合→自动审批流水线
  - 前端 `AIVideoInterviewPage.tsx` 使用 MediaRecorder 录制音频（WebM/Opus格式）
  - 后端 `scoring.routes.ts` 提供3个API端点：`/transcribe-and-score`、`/session/:sessionId`、`/aggregate/:sessionId`
  - 每题独立评分，支持维度评分（dimensionScores）、评分理由（scoreReasoning）
  - 面试完成后自动聚合到 `interview_results` 并创建审批请求
- **面试答题评分表** (`interview_answer_scores`)：记录每道题的音频时长、转录文本、评分、维度分数、使用的LLM模型/提供商
- **JWT Token 黑名单** (`token_blacklist`)：支持 Token 主动吊销，登出后 Token 立即失效
- **审计日志表** (`audit_logs`)：记录用户操作（增删改），包含 action、resource_type、resource_id、IP地址、User-Agent
- **面试模板评分配置**：新增 `scoring_config`（维度+基础分+要求）和 `grade_rules`（分数→等级映射）JSONB字段
- **面试问题扩展字段**：`group_name`（分组）、`follow_ups`（追问）、`scoring_guide`（评分标准表）、`linked_dimensions`（关联维度）
- **Edge Functions 扩展**：从4个扩展到13个（新增 candidates/positions/projects/interviews/approvals/shortlist/outreach/agents），共享模块从2个增加到8个

### 变更
- 数据库迁移从 019 扩展到 025（新增 020-025 六个迁移文件）
- 面试结果表 (`interview_results`) 新增 `question_answers JSONB` 字段存储聚合的答题摘要
- 岗位详情表 (`position_details`) 新增 `profile_rules` 字段（新格式画像配置）
- 生产环境新增索引优化（migration 024）
- 外联模块简化（migration 025）
- 前端环境变量新增 `VITE_SUPABASE_URL`、`VITE_SUPABASE_ANON_KEY`
- 后端环境变量新增 AI 提供商配置（`AI_PROVIDER`、`AI_MODEL_NAME`、`AI_API_KEY`、`AI_BASE_URL`）
- 用户角色从 interviewer 更正为 hiring_manager

## [1.1.0] - 2026-05-11

### 新增
- **LLM Vision 简历解析兜底**：PDF简历解析新增第4级降级链路。当 pdftotext、OCR (Tesseract)、MinerU API 均无法提取文字时，自动将PDF页面转为图片发送给视觉大模型（GPT-4V/Claude/Gemini等）进行文字提取。需在 `ai_model_configs` 表中配置默认的视觉模型

### 变更
- `server/src/modules/ai/llmClient.ts` 新增 `callVisionLLM()` 函数和 `ContentPart` 类型，支持多模态（文字+图片）LLM请求
- `server/src/modules/ai/promptBuilder.ts` 新增 `buildResumeVisionSystemPrompt()` 和 `buildResumeVisionUserMessage()` 提示词
- `server/src/shared/pdfProxy.ts` 重构解析链路：MinerU失败后不再直接报错，改为继续尝试LLM Vision兜底

## [1.0.0] - 2026-05-06

### 新增

#### 核心业务模块
- **项目管理**：项目创建、编辑、状态管理（筹备中/进行中/已关闭），关联岗位和团队
- **岗位标准配置**：画像配置（关键词+同义词+类别）、评分标准（维度+权重+关键字）、Grade Rules（分数档位）、基础分配置
- **候选人搜索**：简历导入（MD格式）、MinerU简历解析、AI智能评分（多维度）、评分详情预览、匹配关键字分析
- **入围名单**：高优先级候选人集中管理、按岗位分组、推进到联系人、发送面试邀请
- **联系人管理**：联系记录CRUD、渠道跟踪（微信/邮件/电话）、状态流转（待处理→已联系→已回复→安排面试→已录用→已拒绝）
- **审批中心**：面试结果审批流程（待审批→通过/驳回）、审批历史记录

#### AI面试中心
- **面试配置**：面试模板CRUD、关联岗位下拉选择、题目管理（标题/内容/时限）、批量保存题目
- **面试管理**：面试会话列表、状态筛选（待开始/进行中/已完成）、进度显示
- **面试结果**：结果列表、详情弹窗（候选人信息/维度得分/等级评定）、CSV导出
- **数据分析**：面试总览统计、分数分布图、月度通过率趋势、按岗位分析

#### 系统功能
- **用户认证**：JWT登录、角色权限控制（admin/recruiter/interviewer/viewer）
- **系统设置**：账户管理、团队邀请、角色权限配置、AI模型配置（支持OpenAI/Anthropic/Gemini/DeepSeek等8个提供商）、通知设置
- **人才库**：候选人数据库管理、标签系统、按项目/岗位分组查看
- **工作台**：数据概览、最近活动、快捷操作入口

#### 评估报告
- **PDF导出**：使用jsPDF直接生成评分报告（含候选人信息/总分/维度得分/关键字匹配）
- **PNG导出**：使用Canvas API生成专业评估卡片（渐变头部/分数圆环/维度卡片/关键字标签）
- **简历下载**：支持原始简历文件下载（PDF/图片格式）
- **外联邮件附件**：生成.eml MIME格式邮件文件，携带简历或评估报告附件

### 修复
- 修复入围名单详情页空白问题（迁移CandidateSearchPage的完整候选人数据加载逻辑）
- 修复候选人详情预览中`profileKeywords`未定义导致崩溃的问题
- 修复RadarChart `dataKey`不匹配导致图表不渲染的问题
- 修复`html2canvas`与Tailwind CSS v4的`oklch()`颜色函数不兼容导致PNG导出失败的问题
- 修复AI面试中心四个模块在真实API模式下字段不匹配的问题（snake_case ↔ camelCase映射）
- 修复面试模板详情API返回flat结构但前端期望嵌套结构的问题
- 修复面试结果中grade字段格式不匹配的问题（A/B/C → excellent/good/qualified/pending/rejected）
- 修复面试模板创建对话框中岗位ID无法选择的问题（改为下拉选择）
- 修复面试配置页「新建面试模板」按钮被底部操作栏遮挡不可见的问题

### 变更
- 移除html2canvas依赖，改用Canvas API和jsPDF直接生成导出文件
- 候选人类型新增`honors`字段（荣誉证书）
- 面试API层新增6个mapper函数，统一处理数据库字段名和前端字段名的转换
