# EM-BOX MVP SPEC

状态: DONE_WITH_CONCERNS

## 1. 说明

本规格按 `/product-spec-builder` 产出，用于定义当前代码基础上可推进的最小可上线版本。

约束说明:
- 本规格已对齐 `MVP.md` 中定义的 MVP 目标、必做模块、必须能力和不做范围
- 本规格同时结合当前代码结构、现有页面能力和已完成的模块化改造整理
- 目标不是完整招聘平台，而是先上线一个能演示闭环、能支撑首轮用户验证的最小版本

## 2. 产品定位

EM-BOX 是一个面向具身智能数据采集岗位的 AI 招聘运营系统。  
第一版 MVP 的价值不是“全流程 ATS”，而是验证下面这件事:

**系统是否能把岗位定义、候选人筛选、入围推进、外联、AI 面试和审批串成一条可用闭环。**

## 3. 目标用户

### 3.1 招聘运营
- 创建或查看岗位标准
- 导入候选人并做搜索筛选
- 将候选人加入入围名单
- 发起外联和推进流程

### 3.2 用人经理
- 查看候选人背景、匹配分和 AI 面试结果
- 对候选人给出通过/拒绝意见

### 3.3 管理者
- 查看转化漏斗
- 查看代理效率、外联回复率、审批状态

## 4. MVP 核心闭环

MVP 必须跑通这条链:

1. 进入系统并登录
2. 创建或查看岗位标准配置
3. 导入候选人并完成结构化解析
4. 基于 Fit Score 和筛选条件完成搜索与筛选
5. 将候选人加入入围名单并推进状态
6. 发起或查看 AI 外联
7. 完成 AI 面试模板配置与视频面试体验
8. 在审批中心完成决策
9. 在数据洞察页查看关键转化指标

## 5. 上线范围

### 5.1 P0 必做

#### A. 基础壳层
- 登录页可进入系统
- 侧边导航可稳定跳转
- 路由懒加载可正常工作
- 桌面浏览器下主要页面可完整展示

#### B. 候选人搜索
- 候选人列表展示
- 导入简历弹窗
- 解析结果页
- 候选人详情弹窗
- 基础搜索和筛选 UI
- Fit Score、标签、评级展示
- 支持加入入围名单
- 支持查看推荐理由

#### C. 岗位标准配置
- 岗位配置页可访问
- 岗位基础信息可浏览和配置
- 现有模板/标准可浏览
- 页面无报错、状态可读

#### D. 入围名单
- 展示入围候选人列表
- 展示下一步动作和优先级
- 支持状态标记与批量推进占位

#### E. 外联序列
- 外联活动列表
- 外联模板数据
- 回复率和活动状态展示
- 区分发送状态与回复状态

#### F. AI 面试
- 面试模板页可浏览
- 面试体验页可打开
- 视频面试核心流程可演示
- 面试摘要、评分、评价结论可展示

#### G. 审批中心
- 审批列表可访问
- 审批状态与操作区可浏览
- 审批历史信息可浏览

#### H. 数据洞察
- 项目时间
- 入围率
- 外联回复率
- 面试转化率
- 审批通过率

### 5.2 P1 可延后
- AI 代理配置的真实编辑能力
- 项目管理的真实项目协同逻辑
- 集成管理的真实连接配置
- 数据洞察的导出和时间筛选真实生效
- 审批、外联、入围之间的真实联动
- 操作日志的完整落地

### 5.3 本期不做
- 多租户
- 复杂 RBAC
- 真正的工作流引擎
- 实时消息
- 后端任务编排中心
- 复杂第三方集成
- 生产级监控告警系统
- 高级 BI 报表
- 复杂 AI 代理编排市场
- 过细的项目管理系统

## 7. 当前 MVP 的主要缺口

### 7.1 功能联动仍然偏弱
- 多数页面仍是静态展示或 mock 数据驱动
- 模块之间缺少统一状态流转
- “加入名单 -> 外联 -> 面试 -> 审批”尚未串成真实链路
- 操作日志尚未形成统一机制

### 7.2 部分核心页仍复用旧页面内核
- `PositionConfigRoute` 仍直接复用根目录页面
- `InterviewTemplatesPage` 仍直接复用根目录页面
- `ApprovalsRoute` 仍直接复用根目录页面

### 7.3 上线定义需要收紧
- `AI 代理`
- `项目管理`
- `集成管理`

以上模块现在更适合定位为展示型辅助页，不应被视为首批上线核心价值模块。

## 8. 建议的最小上线版本

### 8.1 对外演示主线
- 登录
- 岗位配置
- 候选人搜索
- 导入解析
- 加入入围名单
- 外联序列
- AI 面试模板与预览
- 审批中心
- 数据洞察

### 8.2 对外演示时弱化的模块
- AI 代理
- 项目管理
- 集成管理

处理方式:
- 保留入口
- 明确标注“即将接入”或“管理视图”
- 不作为演示闭环主路径

## 9. MVP 验收标准

### 9.1 功能验收
- 用户可从登录页进入系统
- 所有导航页面都能正常打开，不能再出现空白页
- 候选人搜索到审批的主路径可连续演示
- 所有 P0 页面无明显布局断裂或阻塞错误

### 9.2 工程验收
- `npm run lint` 通过
- `npm run build` 通过
- 主路径页面支持桌面浏览器
- 主要页面加载时有明确 loading 状态

### 9.3 演示验收
- 可完成一次“岗位定义 -> 候选人筛选 -> 入围 -> 外联 -> 面试 -> 审批”的端到端演示
- 数据洞察页能反映该流程的核心指标

## 11. 结论

当前系统已经具备“做成 MVP 演示版”的基础，但还不等于“可以正式上线”。

正确做法不是继续扩页面，而是:
- 收紧 MVP 范围
- 强化主链路联动
- 把非核心模块降级为辅助展示
- 完成统一 UI 收口和体验修整

如果按本规格推进，下一步最合理的是启动 `/design-brief-builder`，先把最小上线版本的视觉和交互边界定住。
